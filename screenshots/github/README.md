# screenshots/github — github 主题的视觉参照基准

放进你认可的真实截图,给 AI 当观感锚点。每张图都要在下方登记,写清为什么好、学什么、别机械抄什么。否则 AI 容易把截图里的一切,包括无关细节,都当成要复刻的目标。

建议图:

- `issue-list.png`
- `issue-detail-modal.png`
- `settings-section.png`
- `dense-table.png`
- `form-feedback.png`

## 登记格式

每加一张图,复制下面格式补一段。

### 文件名.png

- **示范什么:** 这张图主要展示哪类做法,例如高密度列表 + 克制用色。
- **学这些点:** 具体要学的 2-4 条,例如状态只用小圆点、表头操作区右对齐、行高紧凑但可点区域足够大。
- **别机械抄:** 哪些是这张图的特定内容,不要照搬,例如具体文案、列数、业务字段。
