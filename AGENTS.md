# AGENTS.md — 给 AI 编码助手(Codex 等)的指令

本仓库的 UI 必须遵循一套设计标准(核心原则 + 选定主题)。

## 本项目使用的主题
> 在这里写明,例如:`themes/github`
>
> **当前主题:** `themes/github`

## 改动 UI 前必读
1. `PRINCIPLES.core.md` —— 跨风格通用原则与硬规则
2. `DECISIONS.core.md` —— 通用决策(勿回退)
3. `<当前主题>/tokens.css` —— 颜色 / 间距 / 字号的唯一数值来源
4. `<当前主题>/PRINCIPLES.md` —— 本风格的补充规则
5. `<当前主题>/DECISIONS.md` —— 本风格的决策(勿回退)

## 硬性要求
- 颜色 / 间距 / 字号 / 圆角**只能引用当前主题 `tokens.css` 的变量**,禁止写死 `#hex` 或随手像素。
- **禁止用彩色背景填充卡片或区域**;颜色只用于状态(小圆点 / 小徽标)和主操作按钮。
- 区域靠 1px 边框和留白区分,不靠颜色或阴影。

## 工作方式
- 较大的视觉改动:**先输出审计 + 方案,等我确认再改代码**,不要直接大改。
- 每次改完 UI 自检并在回复里报告:彩色面积 ≤ 8% / 字号 ≤ 6 档 / 字重 ≤ 3 档 / 间距是 4 的倍数且来自 token / 无彩色背景区域 / 灰度测试通过。

## 决策的「反哺分流」(重要)
当我给出新的视觉判断 / 纠正时,按适用范围归档,不要只在对话里说完就算:
- **只跟本项目有关** → 写进本项目根目录的 `DECISIONS.local.md`(如无则新建),**不要**动 design-standard。
- **适用于本风格的所有项目** → 写进 `design-standard/<当前主题>/DECISIONS.md`(或改 `tokens.css`)。
- **适用于所有风格** → 写进 `design-standard/PRINCIPLES.core.md` 或 `DECISIONS.core.md`。
- 凡是写进了 `design-standard/`(共享子模块)的改动,**提交并 push 该子模块**,以便其它项目能拉取更新。
- 拿不准属于哪一类时,先问我,默认按「只跟本项目有关」处理,避免污染通用标准。
