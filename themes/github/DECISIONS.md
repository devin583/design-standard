# 决策日志:github 主题

> 只记录本风格专属的设计判断。通用的放根目录 `DECISIONS.core.md`。
> 格式:`### YYYY-MM-DD — 一句话结论`,最新放最上。

---

### 2026-06-25 — 状态色锁定 5 色 + 中性,不引入第 6 个色相
- **决定:** github 主题语义色锁定 accent(蓝)/success(绿)/attention(黄)/danger(红)/done(紫)5 种 + 中性灰。像「待外部部门 / external_wait」这类原本用 sky/青的状态,并入 accent(蓝);状态之间靠文字标签区分,不靠新色相。
- **原因:** 「颜色极度克制」。多一个色相就多一分玩具感;状态从不只靠颜色编码,标签文字已足够区分。

### 2026-06-25 — 选中态下划线用蓝,不用 GitHub 的橙
- **背景:** 真实 GitHub 当前 tab 用橙色下划线;但本标准核心约定「选中=蓝」。
- **决定:** 选中态(当前 tab / 导航)下划线用 `--color-accent-fg`(蓝),与链接 / 主操作之外的「可点 / 选中」语义统一,不引入橙色。
- **原因:** 整套界面只保留绿(主操作)+ 蓝(可点/选中)两种交互色,比多加一个橙色更克制。若某项目确实要 GitHub 原味橙,只需在该项目局部覆盖,不进主题。

### 2026-06-24 — 「临近截止」徽标去重
- **背景:** 同一信息在人员标题和每一行里各出现一次。
- **决定:** 只在事项行保留一次。
- **原因:** 违反「一致性 / 不重复表达」。
