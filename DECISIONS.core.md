# 核心决策日志(跨风格通用)

> 只记录适用于**所有风格**的设计判断。风格专属的决策放对应 `themes/<主题>/DECISIONS.md`。
> 格式:`### YYYY-MM-DD — 一句话结论`,最新放最上。

---

### 2026-06-24 — 建立「核心 + 多主题」设计标准
- **决定:** 通用原则与硬规则集中在 `PRINCIPLES.core.md`;颜色/字体/圆角等数值按主题拆到 `themes/<主题>/tokens.css`;每个项目选用一个主题。
- **原因:** 把「凭感觉评审」升级为「按标准检验」,并让标准跨项目复用、可演进。

### 2026-06-24 — 概览统计区不用彩色背景(通用)
- **决定:** 任何项目的概览 / 统计卡都用中性底 + 边框,状态只用小圆点或小徽标点缀,不用整块彩色背景。
- **原因:** 违反「颜色克制」。大块彩色背景让专业工具显得像玩具。
