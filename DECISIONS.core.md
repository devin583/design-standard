# 核心决策日志(跨风格通用)

> 只记录适用于**所有风格**的设计判断。风格专属的决策放对应 `themes/<主题>/DECISIONS.md`。
> 格式:`### YYYY-MM-DD — 一句话结论`,最新放最上。

---

### 2026-06-25 — 新增 token:border-emphasis(hover/active 加重边框)
- **决定:** 各主题 `tokens.css` 补 `--color-border-emphasis`(github #afb8c1 / soft gray-300),变量名作为跨主题契约固定。hover、active 等需要比默认边框更重一档时引用它,不再写死深灰 hex。
- **原因:** 工具型界面普遍有「悬停加重边框」需求,之前只能写死;收进 token 后可跟主题换皮,并通过 hardcodedColors 审计。

### 2026-06-25 — 字号只取自 token 档位,小于正文的一律落到 meta
- **决定:** 组件不写死 px 字号,一律引用 `--text-*`。落地时小于 12px 的旧值统一归并到 `--text-meta`(12px),13–15px 归并到 `--text-body`,确保实际档位 ≤ 6,且与 `scripts/audit-font-sizes.mjs` 的主题白名单一致。
- **原因:** 「11/13/15px 各处微调」最容易累计出十几档字号,违反「字号档位 ≤ 6 + 只用 token」。本次在 infodept 项目把 ~13 档收敛为 5 档验证可行。

### 2026-06-24 — 新增 soft 主题并主题化审计配置
- **决定:** 从已验证项目 expat-flow 提炼 `themes/soft`;同时把新增 token 回填到其它主题,并让审计配置支持 `AUDIT_THEME=github|soft`。
- **原因:** soft 与 github 是不同气质的主题,但跨主题变量契约必须一致;审计规则也必须按主题圆角 / 字号档位判断,否则会误报。

### 2026-06-24 — 建立「核心 + 多主题」设计标准
- **决定:** 通用原则与硬规则集中在 `PRINCIPLES.core.md`;颜色/字体/圆角等数值按主题拆到 `themes/<主题>/tokens.css`;每个项目选用一个主题。
- **原因:** 把「凭感觉评审」升级为「按标准检验」,并让标准跨项目复用、可演进。

### 2026-06-24 — 概览统计区不用彩色背景(通用)
- **决定:** 任何项目的概览 / 统计卡都用中性底 + 边框,状态只用小圆点或小徽标点缀,不用整块彩色背景。
- **原因:** 违反「颜色克制」。大块彩色背景让专业工具显得像玩具。
- **补充(2026-06-25):** 兼作筛选 / 切换的 tile(如工作台顶部「已超期 / 临近截止 / 停滞」)同样算统计卡——颜色只落在左侧小图标方块,卡片本体中性,选中态用蓝色边框。常见坑:tile 复用了通用 `.tone-*` 徽标类,徽标底色把整卡染色;需用更高优先级(如 `[class*="tone-"]`)把卡片复位为中性。infodept 项目已按此修正。
