# 核心设计原则(跨风格通用)

> 这一层定义「什么是好工具」的通用规律,几乎适用于任何专业风格。
> 具体的颜色 / 字体 / 圆角数值由各**主题**决定,见 `themes/<主题>/`。
> 目标参照:GitHub / Linear 那种简洁、克制、高信息密度的工具感。

---

## 六条原则

### 1. 颜色极度克制
界面的 ~90% 由黑 / 白 / 灰构成。颜色只允许用在两类东西上:**有意义的状态** 和 **页面主操作按钮**。
颜色必须小面积(状态点、细徽标、一个按钮)。**禁止用彩色背景填充卡片或区域**,区域靠边框和留白区分。

### 2. 信息密度优先
同样一屏承载更多有效信息。留白用来**分组**,不用来装饰。概览区不该占掉半屏,大数字要克制。

### 3. 层级靠排版,不靠色块
用字号、字重、分隔线、留白建立层级。判断方法:**界面变灰度后层级是否依然清晰**?一变灰就乱,说明在用颜色代替排版。

### 4. 容器样式克制
细边框、小圆角、几乎无阴影。卡片不要「鼓」「软」。只有浮层(弹窗、下拉)才允许一层很淡的阴影。

### 5. 一致性
同一类元素只用一套样式。同一条信息不重复表达。

### 6. 只用 token
所有颜色 / 间距 / 字号 / 圆角只能引用当前主题的 `tokens.css` 变量,禁止写死 `#hex` 或随手像素值。

---

## 可检验的硬规则(改完自检)

- [ ] **彩色面积 ≤ 8%**(任意一屏,非黑白灰像素)
- [ ] **字号档位 ≤ 6**,正文统一
- [ ] **字重档位 ≤ 3**
- [ ] **所有间距是 4 的倍数**且取自 token
- [ ] **颜色只编码状态或主操作**,没有彩色背景的区域 / 卡片
- [ ] **同一条信息只表达一次**
- [ ] **灰度测试通过**
